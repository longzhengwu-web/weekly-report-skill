# 输出风格预设（生成正式报告前先让用户选）

正式生成前，**用同一个代表性条目渲染下面 3 种风格各一段做 demo**，再用 AskUserQuestion 让用户选；
默认值 = `learned_preferences.md` 里记录的上次选择（没有则「详细版」）。选定后记回 learned_preferences。
三种风格都遵守共同底线：结论 > 数量、公司/个人分开、保留英文术语、不出现代码字段、对外脱敏、末尾「风险同步」。

---

## 风格 A：详细版（默认）
结构：总结(1 句) → 本周完成(按主题分小节，每条≈3 句：背景一句 + 产出/结论含指标 + 下一步；可带表格) →
进行中 → 下周重点 → 风险同步。适合正式周报、给 leader 当面汇报。

**demo（一条）：**
> ### 文件类多级缓存 L2 降本（主线）
> 背景：文件类记忆（笔记/网盘/附件）占明细成本较高。本周敲定并实现两级缓存（含修复一处版本未进 key 致
> 升级后缓存永久失效的 bug），上线后实测合计命中率约 N%（L1 约 X%），节省约 XX 万 token，仍约 N 成请求重算。
> 相关：[MR#xxx](…)　下一步：用累计标量补全总节省 token；评估旧记录回填。

## 风格 B：简约版（每件事一句话：因为→做了→结果）
结构：公司/个人各一组单行条目；每条一句话讲清「因为什么 → 做了什么 → 结果如何（带关键指标）」；
末尾「风险同步」每条一句。适合日报、快速同步。

**demo（一条）：**
> - **文件类多级缓存 L2**：因文件类记忆占成本高，本周敲定并上线两级缓存并修复版本失效 bug，实测命中率约 N%、
>   省约 XX 万 token（仍约 N 成重算），下一步补硬数字。

## 风格 C：要点速览版（IM/群消息友好）
结构：极短 bullet，无背景、无小标题段落；每条 `项目 → 结果/指标；下一步`。风险用一行「⚠ 风险：…」。
适合贴进群、站会、status 频道。

**demo（一条）：**
> • 多级缓存 L2 上线 → 命中率约 N%、省约 XX 万 token；下一步补硬数字。

---

可按需扩展更多风格（如"叙事段落版""中英双语版"），用户提出就加进本文件并记入 learned_preferences。
