# 日报 Prompt

先读 `report_principles.md`（三段式、结论>数量、覆盖工作内容/进度/风险）。日报在此
基础上加：**单日粒度**。

## 输入
某一天的会话采集结果：`days[该日期]` 下的若干 session 桶，每个含 `cwd`、
`user_prompts`（意图）、`assistant_snippets`（结果）、`tool_activity`（工作量）、
`files_touched`（改了什么）。

## 处理
1. 把当天的会话按**语义聚成若干工作条目**——同一项目/任务的多个会话合并为一条，
   不要一个会话写一条流水。
2. 每条按三段式写：做了什么 → 产出/结论 → 下一步。
3. 当天的状态变化、阻塞、待确认显式写出。
4. 按重要度排序，噪音（纯配置、无产出的零散查询）合并或丢弃。

## 输出结构（Markdown，给人读）

措辞专业、保留英文术语、不出现代码字段、不用 emoji。分**公司日报**和**个人日报**两份**独立**文档。
**按事件聚合**（同一件事的不同阶段合并成一条，不按会话拆），每条密实、约 3 句。

**日报只要「今日完成」一个板块**。**不要「进行中」「下周重点」「风险同步」这些独立板块——那是周报的**。
卡点/风险/待确认直接并进对应条目里说一句即可。
**日报的条目不要标「主线/调研/支线」**——分层是周报的，日报按重要性排序即可，不加层级标签。

**日报也分风格**（生成前同样让用户选，见 `styles.md`）：**正常版**每条 2–3 句（做了什么+结论/指标+下一步）；
**简洁版**每条一行（做了什么+结论/关键指标，下一步省略或并一短句）；**要点版**纯 bullet。下面是正常版单份模板：

```markdown
# 日报 · {日期}

## 今日完成
### \<事项（保留英文术语）\>
做了什么 + 关键结论/指标；下一步：…（卡点/风险并进这里）。相关：[MR/Notion](…)（如有）

### \<下一个事项\>
…
```

公司或个人当天无内容则省略整份。

## 出完报告后：主动询问归类

先**自己按 `project_categories.md` 预判**好公司/个人与主线/支线分层、给出报告，
再**主动用 AskUserQuestion 问用户**确认或调整（归属 + 分层），回答后写回该表与记忆。

## 自检
- [ ] 每条都有"产出/结论"，不是只有"做了 N 件/N 条"？
- [ ] 进度状态清楚（推进中/完成/阻塞/待确认）？
- [ ] 风险单列？
- [ ] 没有杜撰、没有暴露 session_id/路径？
- [ ] 当天无活动则说明并返回空？
