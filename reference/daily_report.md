# 日报 Prompt

先读 `report_principles.md`（三段式、结论>数量、覆盖工作内容/进度/风险）。日报在此
基础上加：**单日粒度**。

## 输入
某一天的会话采集结果：`days[该日期]` 下的若干 session 桶，每个含 `cwd`、
`user_prompts`（意图）、`assistant_snippets`（结果）、`tool_activity`（工作量）、
`files_touched`（改了什么）。

## 处理
1. 把当天的会话按**语义聚成若干工作条目**——同一项目/任务的多个会话合并为一条，
   不要一个会话写一条流水。
2. 每条按三段式写：做了什么 → 产出/结论 → 下一步。
3. 当天的状态变化、阻塞、待确认显式写出。
4. 按重要度排序，噪音（纯配置、无产出的零散查询）合并或丢弃。

## 输出结构（Markdown，给人读）

**极简**（见 `report_principles.md`"篇幅"）：措辞专业、保留英文术语、不出现代码字段，
**不要"今日概览"、不要 emoji**。每条**约 3 句话**。分**公司/个人**、按**主线/调研/支线**分层、
**按事件聚合**（同一件事的不同阶段合并成一条，不按会话拆），末尾「风险同步」。

```markdown
# 日报 · {日期}

## 主线任务
- **\<事项（保留英文术语）\>**：做了…，\<核心结论+关键指标\>。下一步\<具体事项\>。

## 支线 / 协作类
- **\<事项\>**：…（约 3 句）

## 风险同步
- **\<风险点\>**（进度/延期/依赖/待确认/质量/成本/安全）：什么有风险 + 影响 + 应对。
- …（无则"今日无重大风险"）
```

某层当天无内容则省略整层。风险既在对应条目点到，也在末尾「风险同步」汇总。

## 出完报告后：主动询问归类

先**自己按 `project_categories.md` 预判**好公司/个人与主线/支线分层、给出报告，
再**主动用 AskUserQuestion 问用户**确认或调整（归属 + 分层），回答后写回该表与记忆。

## 自检
- [ ] 每条都有"产出/结论"，不是只有"做了 N 件/N 条"？
- [ ] 进度状态清楚（推进中/完成/阻塞/待确认）？
- [ ] 风险单列？
- [ ] 没有杜撰、没有暴露 session_id/路径？
- [ ] 当天无活动则说明并返回空？
